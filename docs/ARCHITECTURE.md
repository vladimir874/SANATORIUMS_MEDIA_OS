# Архитектура финальной тестовой демо-версии

## Слои системы

1. **SANATORIUMS MEDIA OS** хранит стандарты съемки, роли, контроль качества и
   вспомогательный Google Sheets-процесс.
2. **HOTELCUT Core** локально сканирует медиа и нормализует технические данные.
3. **Processing Pipeline** будет добавляться строго по этапам: прокси, сцены,
   техбрак и монтажный manifest.
4. **Premiere Adapter** в будущем читает manifest и строит таймлайн через UXP.
   Это тонкий интеграционный слой, а не место для анализа видео.

```text
Папка отеля
    -> scanner.py (1.1)
    -> scan manifest
    -> metadata.py + ExifTool (1.2)
    -> enriched manifest
    -> FFmpeg proxies (1.3, следующий шаг)
    -> scenes/QC (этап 2)
    -> edit manifest (этап 3)
    -> Premiere UXP adapter (будущий smoke test)
```

## Неподвижные правила

- Исходники только читаются.
- Идентификатор медиа детерминирован; изменение содержимого ловит sampled SHA-256.
- Входная структура папок классифицируется конфигурацией, а не жестким кодом.
- `capture_order` не равен творческому `edit_order`.
- Неизвестная временная зона не додумывается.
- Смешанные 50 и 60000/1001 fps не приводятся молча к одной частоте.
- Каждый этап имеет отдельные тесты и принимается до начала следующего.

## Формат интеграции с Premiere

Исторический migration pack предлагал FCP XML v1.5. Для Premiere Pro это не
доказанный прямой транспорт: FCPXML и legacy XMEML — разные форматы. Поэтому
основной проектный контракт — версионированный JSON manifest и UXP-адаптер.
XML может появиться как дополнительный экспорт только после отдельного smoke
test в целевой версии Premiere.

## Границы текущей версии

В 0.2.0 нет FFmpeg-прокси, PySceneDetect/OpenCV, автоматического выбора дублей,
генератора таймлайна, UXP-панели и аудита Premiere `project.json`. Наличие
документации об этих блоках не означает, что они реализованы.
