# ИНСТРУКЦИЯ И БАЗА ЗНАНИЙ ДЛЯ GPT-5.6 SOL (MIGRATION PACK: HOTELCUT MVP 2026)

Этот документ предназначен для загрузки в базу знаний (Knowledge) вашего Custom GPT или для отправки в ChatGPT в режиме Work/Codex. Он содержит жесткие архитектурные правила, структуру данных и выводы, полученные по результатам реального успешного прогона MVP HOTELCUT на пакете из 134 исходных файлов.

---

## 1. Системные требования и технический стек MVP (2026)

*   **Язык программирования:** Python 3.10–3.12.
*   **Чтение метаданных:** `ExifTool` (через Python-обертку `PyExifTool`) для моментального анализа кодеков, разрешения и времени съемки на этапе импорта.
*   **Анализ сцен (B-Roll):** `PySceneDetect` (кастомное решение на базе OpenCV) для автоматического разделения исходников на планы.
*   **Автостабилизация:** Библиотека `vidstab` (интегрированная в Python-пайплайн) для выравнивания кадров с сильной тряской (сохраняя оригинальную аудиодорожку).
*   **Формат экспорта таймлайна:** Final Cut Pro XML (FCP XML v1.5). Это единственный надежный способ перенести многодорожечный таймлайн с абсолютными путями, маркерами и отключенными дорожками в Adobe Premiere Pro без зависания интерфейса.

## 2. Жесткие рамки MVP (Ограничение скоупа по результатам тестов)

По согласованию с командой, для ускорения разработки и минимизации ИИ-ошибок, из первой версии MVP **полностью исключены следующие функции**:
*   ❌ **Ретроспективная расшифровка речи и WhisperX внутри ИИ-кода** (все интервью и начитки диктора обрабатываются препроцессором).
*   ❌ **Автоматическое нахождение визуальных дубликатов** на этапе сборки.
*   ❌ **Автоматическая ИИ-сортировка кадров по зонам отеля** (заменена строгой структурой папок, раскладываемых оператором).
*   ❌ **ИИ-выбор лучших дублей речи** (выполняется человеком вручную перед запуском генерации проекта).

## 3. Алгоритм монтажа и логика сборки таймлайна

Система должна строго руководствоваться следующими правилами при генерации FCP XML:

1.  **Порядок сцен:** Сборка таймлайна выполняется строго по утвержденной структуре Sanatoriums.com:
    $$\text{Экстерьер} \rightarrow \text{Лобби/Рецепция} \rightarrow \text{Номера} \rightarrow \text{Питание} \rightarrow \text{Лечение} \rightarrow \text{Wellness} \rightarrow \text{Финал}$$
2.  **Порядок файлов (Запрет на автоперетасовку):** Кадры внутри каждой секции должны располагаться **строго в хронологическом порядке их съемки** (согласно EXIF-дате). Случайная перетасовка ИИ запрещена.
3.  **Длина клипов:** Нарезка планов не должна быть слишком короткой. Длина клипов автоматически растягивается (в среднем около 4 секунд на план), чтобы вся нарезка органично тянулась к целевой длине каждой секции.
4.  **Монтаж начитки и подводок (A-Roll):**
    *   Все файлы подводок и начитки ведущего импортируются на таймлайн **целиком**.
    *   Они размещаются на отдельной **видеодорожке V2**.
    *   **Дорожка V2 по умолчанию выключается (disabled)**, чтобы не мешать монтажеру просматривать черновую склейку B-roll, но сохраняет все исходные файлы.
    *   На файлы начитки накладываются клип-маркеры двух типов:
        *   `Начитка ★` — для качественных реплик (лучших дублей). Текст реплики записывается в комментарий маркера.
        *   `Чернуха` — для технической болтовни съемочной группы, вздохов и пауз (маркер оставляется без текста, чтобы монтажер мог быстро пропустить этот участок).

## 4. Стандартизированная структура папок проекта (2026)

Исходные материалы по каждому отелю должны быть организованы на жестком диске по следующему шаблону:
```text
[Год] > [СТРАНА] > [ГОРОД] > [НАЗВАНИЕ_ОТЕЛЯ]
├───DRON                             # Съемка с воздуха
│   ├───foto
│   └───video
├───Экстеръер + Интеръер             # Фасад, лобби, рецепция, коридоры
├───Двухместный номер                # Жилой фонд (обработка строго пакетом ФОТО+ВИДЕО)
├───Питание                          # Ресторан, шведская линия
│   ├───завтрак
│   ├───обед
│   └───ужин
├───Лечение                          # Процедуры, кабинеты, бальнеология
├───Бассейн / wellness               # Бассейн, сауны, зоны отдыха
├───врач                             # Интервью с врачами
├───повар                             # Интервью с шеф-поваром/диетологом
├───директор                         # Интервью с руководством
├───комменты                         # Видео отзывов гостей
├───proxies/                         # [Генерируется ИИ] Легкие прокси 1280px с поднятыми тенями
└───outbox/                          # [Генерируется ИИ] Файлы экспорта
    ├───project.json                 # Отчет о прогоне (ошибки, брак, нехватка кадров)
    ├───hotelcut-premiere.xml        # Секвенция с привязкой к тяжелым RAW-оригиналам
    └───hotelcut-premiere-proxy.xml  # Секвенция с привязкой к легким прокси из proxies/
```

## 5. Выходные форматы данных и интеграция

### А. Структура отчета `project.json`
По результатам анализа папок система должна сформировать отчет о техническом состоянии материалов и дефиците обязательных кадров:
```json
{
  "project_name": "OLYMP new video",
  "status": "success",
  "total_files_analyzed": 134,
  "timeline": {
    "clips_count": 109,
    "total_duration": "07:34",
    "average_clip_duration": "4.1s"
  },
  "technical_issues": {
    "stable_failures": ["IMGV0307.MP4"],
    "focus_failures": []
  },
  "missing_required_assets": {
    "count": 5,
    "missing_list": [
      "wellness/sauna_wide_shot",
      "treatment/baths_interior",
      "dining/lunch_buffet",
      "exterior/dron_evening",
      "room/bathroom_details"
    ]
  }
}
```

### Б. Спецификация генерации FCP XML v1.5
При написании кода для экспорта таймлайна ИИ должен использовать следующий шаблон структуры XML:
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE xcut SYSTEM "http://apple.com/dds/dtd/xcut-1.5.dtd">
<xcut version="1.5">
  <sequence id="hotelcut-sequence">
    <name>HOTELCUT_DRAFT_TIMELINE</name>
    <duration>454</duration> <!-- 07:34 -->
    <rate>
      <timebase>25</timebase>
      <ntsc>FALSE</ntsc>
    </rate>
    <media>
      <video>
        <!-- Дорожка V1: Автоматическая нарезка B-Roll из папок отеля -->
        <track>
          <clipitem id="clip-1">
            <name>IMGV0301.MP4</name>
            <file id="file-1">
              <pathurl>file://localhost/absolute/path/to/media/2026/POLAND/Swinoujscie/HAMILTON/proxies/IMGV0301_proxy.mp4</pathurl>
            </file>
            <in>0</in>
            <out>100</out>
          </clipitem>
        </track>
        <!-- Дорожка V2: Исходная начитка ведущего (ПО УМОЛЧАНИЮ ВЫКЛЮЧЕНА) -->
        <track enabled="FALSE">
          <clipitem id="voiceover-1">
            <name>1-5.MP4</name>
            <file id="file-vo-1">
              <pathurl>file://localhost/absolute/path/to/media/2026/POLAND/Swinoujscie/HAMILTON/proxies/1-5_proxy.mp4</pathurl>
            </file>
            <!-- Разметка лучших дублей клип-маркерами -->
            <marker>
              <name>Начитка ★</name>
              <in>150</in>
              <out>400</out>
              <comment>Мы находимся на курорте Свиноуйсце, Польша, и сейчас проведем для Вас видео экскурсию...</comment>
            </marker>
            <marker>
              <name>Чернуха</name>
              <in>401</in>
              <out>600</out>
            </marker>
          </clipitem>
        </track>
      </video>
    </media>
  </sequence>
</xcut>
```
